# Bank Marketing Campaign: Term Deposit Prediction

##  Описание бизнес-задачи
Цель проекта — разработать ML-модель для предсказания того, откроет ли клиент банка срочный депозит по итогам телефонной маркетинговой кампании. Модель позволяет оптимизировать работу колл-центра, отсеивая "холодную" базу и фокусируя менеджеров на наиболее перспективных лидах.

##  Стек технологий и архитектура
* **Язык и данные:** Python, Pandas, Plotly (EDA)
* **Машинное обучение:** Scikit-Learn, XGBoost
* **Архитектура кода:** Использование `Pipeline` и `ColumnTransformer` для защиты от утечек данных (Data Leakage)
* **Оптимизация гиперпараметров:** байесовская оптимизация через **Optuna**

##  Ключевые метрики
Так как классы были сбалансированы, использовалась связка Accuracy и ROC-AUC (для оценки качества ранжирования клиентов).
* **Baseline (Константная модель):** ROC-AUC = 0.5000
* **Baseline (Logistic Regression):** ROC-AUC = 0.9038, Accuracy = 0.8128
* **Финальная модель (XGBoost + Optuna):** ROC-AUC = 0.9233, Accuracy = 0.8442
> *Примечание: разница метрик между кросс-валидацией (0.9258) и тестовой выборкой (0.9233) минимальна, что говорит о высокой стабильности модели и отсутствии переобучения.*

##  Бизнес-инсайты (Feature Importance)
При интерпретации модели XGBoost выявлены следующие закономерности:
1. **Лояльность клиента:** Сильнейшим предиктором является успех предыдущих маркетинговых кампаний (`poutcome_success`).
2. **Фактор длительности звонка:** Признак `duration` имеет огромный вес, подтверждая правило продаж "чем дольше диалог, тем выше шанс сделки". Однако, для внедрения модели в production (скоринг до начала звонка) этот признак был проанализирован и подготовлен к исключению.
3. **Качество базы:** Звонки на неизвестные типы контактов (`contact_unknown`) резко снижают вероятность конверсии.
